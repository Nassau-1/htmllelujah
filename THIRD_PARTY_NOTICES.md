Canonical license texts are distributed with installed packages. Release builds
must include all notices required by their resolved dependency graph.

This is an initial direct-dependency inventory, not a release SBOM. Before any
binary distribution, the locked direct and transitive graph must be scanned,
reviewed, and recorded in the CycloneDX SBOM generated by pnpm. This file must be
updated in the same change that adds or upgrades a dependency.

The locked development graph also contains build-only packages under
BlueOak-1.0.0, CC-BY-3.0, and MPL-2.0. They have been reviewed as toolchain-only:
none is a runtime dependency of the desktop application or document model. The
CC-BY SPDX metadata remains attributed in its installed package. The automated
license check fails if any of these licenses enters a production dependency graph.

The default allowlist for code dependencies is MIT, ISC, BSD-2-Clause,
BSD-3-Clause, Apache-2.0, and Zlib. CC0 assets and OFL fonts may be accepted with
their required notices. Any other license requires a documented review before use.
