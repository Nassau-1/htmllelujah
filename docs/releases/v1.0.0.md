## Collaboration and synchronized-folder boundary

- One authenticated LAN session has one authoritative host and one in-session
  shared-file writer.
- The text inspector requests, renews, releases, and displays an expiring element lock;
  another participant sees the owner state and cannot edit that element directly.
- A writer sidecar is enforceable across machines only when all participants observe
  one coherent filesystem namespace, such as SMB/NAS.
- OneDrive, Google Drive, Dropbox, and comparable synchronized folders expose local
  replicas. They transport `.hdeck` snapshots but cannot prevent two independently
  started hosts. Participants must designate one host and join that LAN session.
- V1 has no internet relay, automatic writer election, disconnected edit queue, or
  offline merge.

## Residual limitations for release notes

- Windows 11 x64 only; the editor requires a workspace at least 1080 CSS pixels wide.
- The installer may be unsigned and trigger a Windows reputation warning; exact state
  is **PENDING** above.
- Exported HTML is output only. There is no third-party presentation-file interchange.
- Caret-range mixed-run formatting, transitions, animations, notes, comments, linked
  charts, formulas, hosted templates, cloud accounts, relay collaboration, embedded
  model inference, advanced freeform paths, and boolean vector operations are outside
  V1.
- Future unknown `.hdeck` schema versions are refused with a typed compatibility error
  and left unchanged; preview-only opening is post-V1.
- Consumer cloud-sync replicas do not provide a cross-machine writer lock.
- Narrator/NVDA spoken-output validation remains **PENDING** unless filled above.
